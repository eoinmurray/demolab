Most figures in this lab are *generated* тАФ a tool runs an experiment, matplotlib renders a `.png`, and the notebook runner copies it into the post. That's the right path for anything that's the output of a computation.

But sometimes you want a **hand-built diagram** тАФ a schematic, a geometric illustration, a figure that isn't the result of a simulation. Because posts are MDX, you can draw these inline as **SVG** and drive them with `export const` values, so the whole figure stays parametric: change one constant and every coordinate follows. No image files, no build step, and it scales crisply.

Here's the technique with a worked example, then a look at how it's wired up.

## A worked example: a visual proof of Pythagoras

Draw a square of side $a + b$. Inside it, mark a point on each edge that divides the edge into a segment of length $a$ and a segment of length $b$, going around in the same rotational sense. Connect those four points. You get a tilted square whose side is the hypotenuse $c$ of a right triangle with legs $a$ and $b$, and four congruent copies of that triangle filling the corners.

export const a = 120;
export const b = 160;
export const s = a + b;
export const stroke = "#444";
export const triFill = "#cfdcef";
export const cFill = "#c4dcb6";
export const aFill = "#e8b4b4";
export const bFill = "#e6cf99";
export const label = { fontFamily: "var(--font-serif)", fontStyle: "italic", fontSize: 18, fill: "#222" };

<div style="display: flex; gap: 1rem; flex-wrap: wrap; justify-content: center; margin: 1.5rem 0;">
  <figure style="margin: 0; max-width: 320px;">
    <svg viewBox={`0 0 ${s} ${s}`} width="280" height="280" xmlns="http://www.w3.org/2000/svg">
      <polygon points={`0,0 ${a},0 0,${b}`} fill={triFill} stroke={stroke}/>
      <polygon points={`${a},0 ${s},0 ${s},${a}`} fill={triFill} stroke={stroke}/>
      <polygon points={`${s},${a} ${s},${s} ${b},${s}`} fill={triFill} stroke={stroke}/>
      <polygon points={`${b},${s} 0,${s} 0,${b}`} fill={triFill} stroke={stroke}/>
      <polygon points={`${a},0 ${s},${a} ${b},${s} 0,${b}`} fill={cFill} stroke={stroke}/>
      <text x={a/2} y={-6} textAnchor="middle" {...label}>a</text>
      <text x={a + b/2} y={-6} textAnchor="middle" {...label}>b</text>
      <text x={(a + s)/2 + 6} y={(0 + a)/2 - 4} {...label}>c</text>
    </svg>
    <figcaption style="text-align: center; font-size: 0.85rem; color: var(--color-muted); margin-top: 0.4rem;">
      Four triangles plus a square of area c┬▓.
    </figcaption>
  </figure>

  <figure style="margin: 0; max-width: 320px;">
    <svg viewBox={`0 0 ${s} ${s}`} width="280" height="280" xmlns="http://www.w3.org/2000/svg">
      <rect x="0" y={a} width={b} height={b} fill={bFill} stroke={stroke}/>
      <rect x={b} y="0" width={a} height={a} fill={aFill} stroke={stroke}/>
      <polygon points={`0,0 ${b},0 0,${a}`} fill={triFill} stroke={stroke}/>
      <polygon points={`${b},0 ${b},${a} 0,${a}`} fill={triFill} stroke={stroke}/>
      <polygon points={`${b},${a} ${s},${a} ${b},${s}`} fill={triFill} stroke={stroke}/>
      <polygon points={`${s},${a} ${s},${s} ${b},${s}`} fill={triFill} stroke={stroke}/>
      <text x={b/2} y={a + b/2 + 6} textAnchor="middle" {...label}>b┬▓</text>
      <text x={b + a/2} y={a/2 + 6} textAnchor="middle" {...label}>a┬▓</text>
    </svg>
    <figcaption style="text-align: center; font-size: 0.85rem; color: var(--color-muted); margin-top: 0.4rem;">
      The same four triangles plus a square of area a┬▓ and a square of area b┬▓.
    </figcaption>
  </figure>
</div>

Both squares have side $a + b$, so they have the same total area. Both contain the same four right triangles. Therefore the parts that are *not* triangles must also have the same area:

$$
c^2 \;=\; a^2 + b^2.
$$

That is the entire proof. No coordinates, no algebraic identities, no trigonometry тАФ just the fact that area is conserved when you rearrange congruent pieces inside a fixed container.

## How the diagram is built

The whole thing is plain SVG with a few conveniences MDX gives you for free:

- **Parametric constants.** `export const a`, `b`, тАж sit at the top of the post and feed every coordinate. `s = a + b` is the square's side; the diagram redraws itself if you change a single number. (Here $a = 120$, $b = 160$ тАФ a $3{:}4$ ratio, so the inner tilted square has side $5$ тАФ but any positive values work.)
- **Expressions in attributes.** Because this is MDX/JSX, every attribute can take a JavaScript expression: the `viewBox`, each polygon's `points`, and label positions like `x={a/2}` are all computed from the constants rather than hard-coded. The `viewBox` makes the coordinate system resolution-independent тАФ `width` and `height` just scale it.
- **Shared style via spread.** `export const label = {тАж}` holds the text styling once, and each `<text {...label}>` spreads it in, so all the labels stay consistent.

The same pattern covers most hand-drawn figures: define your sizes as constants, lay out `<polygon>` / `<rect>` / `<line>` / `<text>` against them, and let the post stay self-contained. Reach for a tool-generated PNG when the figure is *data*; reach for inline SVG when it's a *drawing*.
